# Internationalization

## Problem

This repository targets both global search traffic and Chinese-speaking operators. A single-language repository is harder to discover, read, and reuse across Codex, Claude Code, OpenClaw, and SearXNG communities.

## Approach

Use English as the canonical source and maintain selected Chinese translations for high-value entry points.

Canonical files:

- `README.md`
- `docs/`
- `examples/`
- `templates/`
- `registry/resources.json`

Chinese entry points:

- `README.zh-CN.md`
- `docs/zh-CN/README.md`
- selected translated docs under `docs/zh-CN/`

## Translation Rules

- Keep English files canonical for governance, validation, schemas, and issue flow.
- Keep Chinese translations concise and faithful to the English source.
- Add a language switch near the top of translated entry points.
- Add translated durable resources to `registry/resources.json`.
- Use `language` and `translationOf` metadata when a registry item is translated.
- Do not translate secrets, private endpoints, local paths, or environment-specific details into public docs.

## Search And Discovery

GitHub search can discover both languages when each translation is a real Markdown file with relevant terms.

## Maintenance Model

Each translation should state that English is canonical. If the English source changes, update the translated file in the same PR when practical. If that is not practical, mark the translation as needing refresh in the PR notes.

## Acceptance Criteria

- root README links to the Chinese README
- Chinese README links back to the English README
- translated durable resources are indexed in the registry
- AI agents can filter resources by `language`
- public examples remain privacy-safe in both languages
