# Contributing

## Before You Submit

- Keep changes focused.
- Prefer small, composable contributions.
- Use public-safe examples only.
- Run `make review` before opening a PR.

## Accepted Contribution Types

- best practices
- integration examples
- evaluation methods
- research summaries
- registry entries
- validation improvements

## Content Rules

- do not commit secrets, tokens, cookies, session material, or personal email addresses
- do not include private SearXNG instance details unless they are explicitly public
- keep examples minimal and reusable
- link every registry entry to a real file in the repository

## Pull Requests

State:

- what changed
- why it matters
- acceptance criteria
- privacy considerations
